# AI北海道旅行プランナー — マネタイズ設計書

## サービスコンセプト

> 「北海道、どこ行けばいい？」をAIが3分で解決する

@hokkaido_lovers の13万フォロワーが最も抱える悩み **「北海道は広すぎて、どこに行けばいいかわからない」** を、4,462件の投稿データ（=13万人が「美しい」と評価したスポット情報）をベースにしたAIが一発で解決する。

---

## 収益構造の全体像

このサービスは **「5層の収益レイヤー」** で構成される。単一の課金モデルに依存せず、ユーザーの行動導線の各段階で収益が発生する設計。

```
┌─────────────────────────────────────────────────┐
│                 ユーザーの行動導線                    │
│                                                     │
│  ① Instagramで風景を見る                             │
│      ↓                                              │
│  ② 「北海道行きたい」→ プランナーにアクセス（無料）    │
│      ↓                                              │
│  ③ 条件入力 → AIが旅程を生成                         │
│      ↓                                              │
│  ④ 旅程内の宿・アクティビティを予約（★ 収益発生）     │
│      ↓                                              │
│  ⑤ 旅行後、写真を #hokkaido_lovers で投稿（循環）     │
│                                                     │
│  ──────────── 収益レイヤー ────────────              │
│                                                     │
│  Layer 1: 予約アフィリエイト（メイン収益）             │
│  Layer 2: プレミアムプラン課金（サブスク）             │
│  Layer 3: 観光事業者のスポンサード掲載（B2B）          │
│  Layer 4: ふるさと納税・物販アフィリエイト             │
│  Layer 5: データ分析レポート販売（B2B）               │
│                                                     │
└─────────────────────────────────────────────────┘
```

## Layer 1: 予約アフィリエイト（メイン収益・全体の50〜60%）

### 仕組み

AIが生成する旅程の中に、**宿泊施設・体験アクティビティ・レンタカー・航空券**の予約リンクを自然に組み込む。ユーザーがそのリンク経由で予約すると、アフィリエイト報酬が発生する。

### なぜこれがメイン収益になるのか

1. **ユーザーは「旅行に行く気満々」の状態でサービスを使う**: 一般的なブログアフィリエイトと違い、このサービスのユーザーは「旅行計画のために来ている」ため、予約への転換率が圧倒的に高い
2. **AIが「おすすめの宿」を旅程に組み込む = 最強のネイティブ広告**: 旅程の一部として宿泊施設が提示されるため、広告感がゼロ。「AIがこの場所に最適な宿を選んでくれた」という信頼感で予約される
3. **Cookieの長期間有効**: 楽天トラベルは30日間のCookie有効期間があるため、その場で予約しなくても後日予約されれば報酬が発生する

### 具体的な提携先と報酬率

| 提携先 | 報酬率 | 1予約あたりの報酬目安 | 特徴 |
|--------|--------|---------------------|------|
| 楽天トラベル | 1%（上限1,000円/件） | 300〜1,000円 | Cookie 30日。楽天経済圏ユーザーに強い |
| じゃらんnet | 1〜1.1% | 300〜1,000円 | バリューコマース経由が最適。体験予約も対象 |
| 一休.com | 1〜2% | 500〜2,000円 | 高単価宿泊に強い。単価が高い分、報酬額も大きい |
| Booking.com | 25〜40%（コミッションの） | 500〜3,000円 | インバウンド向け。報酬率が非常に高い |
| じゃらん遊び・体験 | 2〜3% | 100〜500円 | アクティビティ予約。件数で稼ぐモデル |
| レンタカー各社 | 1〜3% | 200〜1,000円 | 北海道旅行にレンタカーは必須。高い予約率 |

### 月間収益シミュレーション

```
前提条件:
- 月間プラン生成数: 5,000件（13万フォロワーの3.8%が月1回利用）
- プランから予約に至る率（CVR）: 8%（旅行目的の高意欲ユーザー）
- 1予約あたりの平均報酬: 700円

計算:
5,000件 × 8% × 700円 = 280,000円/月

成長後（月間20,000件生成時）:
20,000件 × 8% × 700円 = 1,120,000円/月
```

## Layer 2: プレミアムプラン課金（全体の20〜25%）

### フリーミアム設計

| 機能 | 無料プラン | プレミアム（月額980円） |
|------|-----------|---------------------|
| 旅程生成 | 月2回まで | 無制限 |
| 旅程の日数 | 2泊3日まで | 7泊以上対応 |
| スポット情報 | 基本情報のみ | 撮影ベストタイム・穴場度・混雑予測付き |
| ルート最適化 | なし | 移動時間最短のルート自動計算 |
| オフライン保存 | なし | PDF/画像で保存可能 |
| 季節限定スポット | なし | リアルタイムの開花・紅葉・積雪情報連動 |
| 予算最適化 | なし | 予算内に収まるプランを自動調整 |
| 多言語対応 | 日本語のみ | 英語・中国語・韓国語 |

### なぜ無料プランを用意するのか

1. **無料プランの旅程にもアフィリエイトリンクが入る**: 無料ユーザーからもLayer 1の予約アフィリエイト収益が発生するため、無料でも赤字にならない
2. **フリーミアムは「試してから買う」心理に最適**: AI旅行プランナーは新しいサービスカテゴリのため、まず無料で体験してもらうことが普及の鍵
3. **無料ユーザーの口コミがマーケティングになる**: 「このサービスで作ったプランで北海道旅行したら最高だった」というUGCがInstagramで拡散される

### 月間収益シミュレーション

```
前提条件:
- 月間アクティブユーザー: 5,000人
- 有料転換率: 5%（旅行系サブスクの平均的な転換率）
- 月額: 980円

計算:
5,000人 × 5% × 980円 = 245,000円/月

成長後（MAU 20,000人時）:
20,000人 × 5% × 980円 = 980,000円/月
```

## Layer 3: 観光事業者のスポンサード掲載（全体の10〜15%）

### 仕組み

北海道のホテル・旅館・アクティビティ事業者から月額掲載料を受け取り、AIが生成する旅程に「おすすめ」として優先表示する。

### 掲載プラン

| プラン | 月額 | 内容 |
|--------|------|------|
| ライト | 30,000円 | AIの候補リストに優先表示 |
| スタンダード | 80,000円 | 旅程に「編集部おすすめ」として掲載 + 施設詳細ページ |
| プレミアム | 150,000円 | 旅程への優先組み込み + Instagram投稿での紹介1回/月 |

### なぜ事業者が払うのか

1. **「旅行計画中のユーザー」にリーチできる広告媒体は少ない**: Google広告やSNS広告では「今まさに北海道旅行を計画している人」をピンポイントでターゲティングするのが難しい。このサービスのユーザーは全員がそのセグメント
2. **AIからの「おすすめ」は口コミ以上の信頼度**: 「AIが自分の条件に合った宿として選んでくれた」という体験は、通常の広告よりも信頼度が高い
3. **費用対効果の計測が容易**: 掲載経由の予約数・PV数をダッシュボードでリアルタイム表示。ROIが明確

### 月間収益シミュレーション

```
前提条件:
- ライト契約: 10社 × 30,000円
- スタンダード契約: 3社 × 80,000円
- プレミアム契約: 1社 × 150,000円

計算:
300,000 + 240,000 + 150,000 = 690,000円/月

（ただし営業工数が必要なため、Phase 2以降で展開）
```

## Layer 4: ふるさと納税・物販アフィリエイト（全体の5〜10%）

### 仕組み

AIが旅程を生成した後、「この地域のふるさと納税返礼品」「旅行前に準備しておきたいアイテム」をレコメンドする。

### 具体的な導線

```
AI旅程: 「2日目：富良野のラベンダー畑を訪問」
    ↓
レコメンド枠:
  🏠 「富良野市のふるさと納税で人気の返礼品」
      → 富良野メロン（15,000円）→ アフィリエイトリンク
  📷 「ラベンダー畑の撮影におすすめ」
      → PLフィルター（Amazon）→ アフィリエイトリンク
```

### なぜ旅程の後にこれが刺さるのか

1. **「この土地に行く」という感情が最も高まった瞬間に提示される**: 旅程を見て「富良野楽しみ！」と思った直後に富良野のふるさと納税が表示されると、「この町を応援したい」という感情で購入される
2. **旅行に行けない人の代替行動として機能**: プランは作ったが実際には行けないユーザーにとって、ふるさと納税は「旅行の代わりにその地域を楽しむ」手段になる
3. **楽天ふるさと納税の報酬率は1%だが、単価が高い**: 1件15,000円の返礼品で150円。少額だが大量に発生し得る

## Layer 5: データ分析レポート販売（全体の0〜5%・長期施策）

### 仕組み

蓄積されたプラン生成データ（どのエリアが人気か、どの季節に需要が高いか、どの年代がどこに行きたいか）を、匿名化・統計処理した上で、自治体・DMO・観光事業者に販売する。

### レポート例

- 「2026年春、AIプランナー利用者の北海道旅行トレンドレポート」
- 「年代別・北海道の人気エリアランキング（AIプランナーデータ版）」
- 「インバウンド旅行者の北海道旅程パターン分析」

### 価格: 1レポート 50,000〜200,000円

### なぜ買われるのか

このデータは「北海道に行きたい人がAIに本音を入力した」リアルな需要データであり、アンケート調査よりも信頼性が高い。自治体のマーケティング施策や、事業者の商品開発の根拠として活用される。

## 実装ロードマップ

### Phase 0: MVP（最小実行可能版）— 開発期間2〜3週間

**やること**: Webサイトすら作らず、Instagramのストーリーズ＋Googleフォーム＋手動AI生成で需要を検証する。

```
導線:
  Instagram ストーリーズ
    「あなた専用の北海道旅行プランをAIが作ります（無料）」
      ↓
  Googleフォーム（旅行日数・興味・予算を入力）
      ↓
  運営が手動でClaude/GPTに入力→旅程生成
      ↓
  旅程PDF + アフィリエイトリンク付きをDMで送付
```

**目的**:
- 実際に何人が申し込むか（需要検証）
- どんな条件の入力が多いか（プロダクト設計のヒント）
- アフィリエイトリンクのクリック率・予約率の検証

**コスト**: 0円（自分の手動作業のみ）

### Phase 1: Webアプリ化 — 開発期間1〜2ヶ月

**やること**: Phase 0の検証結果をもとに、自動化されたWebアプリを構築。

- Next.js + Supabase + Claude API でWebアプリ開発
- スポットデータベースの構築（投稿データのスクレイピング＋手動補完）
- アフィリエイトリンクの自動挿入ロジック
- Stripe連携でプレミアムプラン課金

**開発**: AIコーディングツール（Claude Code / Cursor等）で自分1人で開発可能

### Phase 2: グロース — 3〜6ヶ月目

- Instagram投稿と連動した自動プロモーション
- SEO対策（「北海道旅行 プラン」等のキーワードで流入獲得）
- 多言語対応（英語・中国語）でインバウンド市場参入
- 観光事業者への営業開始（Layer 3のスポンサード掲載）

### Phase 3: スケール — 6ヶ月目以降

- LINE公式アカウント連携（LINEから旅程生成）
- 旅行中のリアルタイムAIコンシェルジュ機能

## 競合との差別化

### 既存のAI旅行プランナーとの比較

| 項目 | 一般的なAI旅行プランナー | @hokkaido_lovers AI旅行プランナー |
|------|----------------------|-------------------------------|
| データソース | Google Maps / 旅行サイトの一般情報 | **13万人が「美しい」と評価した独自データ** |
| 対象エリア | 全国 or 全世界（広く浅い） | **北海道特化（狭く深い）** |
| スポットの質 | 有名観光地中心 | **穴場含む4,462件の実証済みスポット** |
| 写真情報 | なし or 汎用的 | **実際の投稿写真でイメージが湧く** |
| 撮影ガイド | なし | **ベスト撮影時間・構図・機材のアドバイス** |
| 信頼性 | AI生成の一般情報 | **コミュニティが実際に訪れて評価した場所** |
| 集客チャネル | 広告頼み | **13万フォロワーへの直接リーチ** |

### 最大の堀（Moat）

**「13万人のフォロワーが毎日投稿するUGCデータ」は、競合が絶対にコピーできない資産。** Google や Trip.com がAI旅行プランナーを作っても、「北海道の景色を愛する13万人のコミュニティが選んだスポット」というデータは持っていない。

## リスクと対策

| リスク | 対策 |
|--------|------|
| AI生成の旅程が不正確 | スポットDBを人力で検証。ユーザーフィードバック機能で継続改善 |
| アフィリエイト報酬率の引き下げ | 複数ASPに分散。自社予約機能の検討 |
| 大手（Google/Trip.com）の参入 | 「北海道特化 × コミュニティデータ」のニッチで勝負。大手は汎用的で深さで勝てない |
| 個人情報・UGC写真の権利問題 | 利用規約の整備。写真利用は投稿者の許諾を取得 |
| APIコストの高騰 | Haiku等の低コストモデルを使用。キャッシュ機能で重複リクエスト削減 |
